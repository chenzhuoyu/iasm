UADALP

Unsigned Add and Accumulate Long Pairwise. This instruction adds pairs of adjacent unsigned integer values from the vector in the source SIMD&FP register and accumulates the results with the vector elements of the destination SIMD&FP register. The destination vector elements are twice as long as the source vector elements.

Depending on the settings in the CPACR_EL1, CPTR_EL2, and CPTR_EL3 registers, and the current Security state and Exception level, an attempt to execute the instruction might be trapped.

Uop

UADALP <Vd>.<Ta>, <Vn>.<Tb>

integer d = UInt(Rd); integer n = UInt(Rn); if size == '11' then UNDEFINED; integer esize = 8 << UInt(size); integer datasize = if Q == '1' then 128 else 64; integer elements = datasize DIV (2 * esize); boolean acc = (op == '1'); boolean unsigned = (U == '1');

Operation

CheckFPAdvSIMDEnabled64(); bits(datasize) operand = V[n, datasize]; bits(datasize) result; bits(2*esize) sum; integer op1; integer op2; if acc then result = V[d, datasize]; for e = 0 to elements-1 op1 = Int(Elem[operand, 2*e+0, esize], unsigned); op2 = Int(Elem[operand, 2*e+1, esize], unsigned); sum = (op1+op2)<2*esize-1:0>; if acc then Elem[result, e, 2*esize] = Elem[result, e, 2*esize] + sum; else Elem[result, e, 2*esize] = sum; V[d, datasize] = result;

Operational information

If PSTATE.DIT is 1:

  • The execution time of this instruction is independent of:
    • The values of the data supplied in any of its registers.
    • The values of the NZCV flags.
  • The response of this instruction to asynchronous exceptions does not vary based on:
    • The values of the data supplied in any of its registers.
    • The values of the NZCV flags.
